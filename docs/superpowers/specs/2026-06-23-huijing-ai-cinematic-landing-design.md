# 绘境 AI「暗光剧场」电影式落地页设计规格

日期：2026-06-23

状态：已确认，待实现计划

范围：绘境 AI 营销首页及其必要的前端交互，不包含真实生图、认证、支付或会员后台。

## 1. 目标

将绘境 AI 原有的象牙白编辑室首页升级为一套黑色、强对比、具有电影叙事感的高端落地页。页面不应像普通 AI SaaS 模板，而应像高端电影官网、奢侈品牌广告片与艺术杂志共同构成的数字展映。

核心结果：

- 六个高屏章节，每一屏都可以独立成立为电影海报；
- 通过巨幕影像、投影光束、空间纵深和克制的滚动动画推进叙事；
- 保留产品价值表达，让用户理解绘境 AI 如何把自然语言转化为专业视觉语言；
- 桌面端强调沉浸、尺度和留白，移动端保持竖版电影海报感；
- 页面可本地运行，并具备真实可操作的导航、CTA、滚动定位与 Gallery 悬停反馈。

## 2. 视觉目标

选定方向：`03 / DARK LIGHT / 暗光剧场`

视觉参考图：

`docs/superpowers/specs/assets/huijing-ai-dark-light-reference.png`

该参考图是视觉目标，不是直接作为整页背景使用的成品。实现需要忠实复现其设计语言：

- 近乎纯黑的展映空间；
- 浮动巨幕与低饱和电影画面；
- 冷白投影光束、薄雾和轻微镜面反射；
- 大尺度中文标题与窄体英文技术标签；
- 细线、场记编号、胶片参数与克制的冷青高光；
- 内容主要依靠光、尺度、景深和排版分层，而不是依靠卡片堆叠。

## 3. 设计原则

### 3.1 像展映，不像模板

章节之间不使用常规浅色背景块或重复卡片容器。每一屏应通过光源、图片位置、标题尺度和空间留白形成独立场景。

### 3.2 光负责引导

冷白光束是全页的叙事线索：Hero 中照亮巨幕，流程区连接步骤，最终 CTA 中收束为单一聚光。光效必须克制，不做霓虹灯或科技炫光。

### 3.3 内容始终可读

氛围不能牺牲信息层级。正文和按钮需满足清晰对比，雾、模糊和颗粒不得覆盖关键文案。

### 3.4 动效慢、稳、可中断

动画使用缓慢淡入、轻微位移和有限视差。页面不劫持滚轮、不强制滚动吸附、不使用弹跳、旋转或卡通缓动。

## 4. 视觉系统

### 4.1 色彩

```text
Canvas Black      #030405
Stage Black       #07090B
Graphite          #111419
Smoked Glass      rgba(17, 20, 25, 0.58)
Primary Text      #F2F3F1
Secondary Text    #9A9FA7
Hairline          rgba(255, 255, 255, 0.16)
Cold Highlight    #B9E2F8
Cold Highlight 2  #7CBBD8
```

冷青色只用于当前步骤、细小定位点、按钮边缘高光和局部光束，不应用作大面积背景渐变。

### 4.2 字体

最多使用两套字体：

- 中文展示与品牌宣言：`Noto Serif SC`，用于大标题和关键宣言；
- UI、正文和英文标签：`Noto Sans SC`，英文使用大写和扩展字距。

若线上字体加载失败，分别回退至 `Songti SC / SimSun / serif` 和 `PingFang SC / Microsoft YaHei / sans-serif`。

### 4.3 排版

- 桌面 Hero 主标题使用 `clamp(64px, 8vw, 132px)`；
- 章节标题使用 `clamp(42px, 5.4vw, 84px)`；
- 移动端主标题为 44–58px，允许分为 2–4 行；
- 正文主要为 14–17px，最大行宽约 34 个中文字符；
- 英文标签为 10–12px，使用大写、宽字距和低对比灰；
- 细线宽度统一为 1px，不使用厚重边框；
- 大部分区域保持直角或 2–4px 微圆角，避免柔软的 SaaS 圆角语言。

### 4.4 纹理与光效

- 全局添加低透明度胶片颗粒层；
- 仅在光源附近使用局部模糊和柔和 bloom；
- 背景允许极轻的网格、胶片边缘或投影尘埃；
- 图片统一使用低饱和、深黑遮罩和冷色调色；
- 不使用大面积彩色渐变、彩虹色、紫蓝霓虹或廉价玻璃拟态。

## 5. 页面结构

### 5.1 顶部导航

导航覆盖在 Hero 上方，桌面端保持透明或轻微黑色模糊，滚动离开 Hero 后提高背景不透明度。

内容：

- 品牌：`绘境 AI`
- 锚点：`作品`、`创作流程`、`积分与会员`
- 主入口：`开始创作`

交互：

- 锚点平滑滚动至相应章节；
- `开始创作` 进入 `/generate`；
- 移动端使用简洁菜单按钮展开全屏黑色导航层；
- 导航层支持关闭按钮、点击链接关闭和 Escape 关闭。

### 5.2 第一屏：Hero / FRAME 01

目标：让用户进入一间黑色放映厅，并立即理解产品定位。

布局：

- 最小高度为 `100svh`；
- 左侧或左下为标题与 CTA；
- 右侧或中央为倾斜的巨大电影画面；
- 使用一大一小两个人物剪影建立空间尺度；
- 标题与巨幕之间保留充足黑色空间。

文案：

```text
FRAME 01 / VISUAL ENGINE / AI IMAGE STUDIO

像导演一样
生成你的视觉大片

从灵感、风格、主体到画面语言，
让 AI 自动理解你的创作意图。

开始创作
查看示例
```

交互：

- `开始创作` 进入 `/generate`；
- `查看示例` 滚动至 Gallery；
- Hero 文案首次进入时进行电影字幕式淡入；
- 背景巨幕与前景剪影产生不同速率的轻微视差；
- 向下滚动时标题缓慢上移并降低透明度。

### 5.3 第二屏：核心能力 / FRAME 02

标题：

`不是输入提示词，而是构建一场视觉叙事`

三项能力：

1. `选择图片类型`：海报、人像、产品、建筑、概念艺术等；
2. `输入主体与需求`：使用自然语言描述想法、用途与气氛；
3. `自动生成专业提示词`：系统将普通描述转化为专业视觉语言。

视觉：

- 三块高而窄的半透明“光幕”排列在空间中；
- 光幕可有细边框和局部顶光，但不能像普通功能卡片；
- 桌面端通过轻微透视、错位和遮挡表现纵深；
- 移动端取消透视，改为单列分镜。

动画：

- 三块光幕依次淡入并上移 18–28px；
- 当前可视光幕的顶部冷光略微增强；
- 动画仅播放一次，回滚不重复闪烁。

### 5.4 第三屏：创作流程 / FRAME 03

标题：

`从想法到大片，只需五个镜头`

流程：

1. `STEP 01 / MOOD` — 选择风格；
2. `STEP 02 / SUBJECT` — 输入主体；
3. `STEP 03 / DETAILS` — 补充细节；
4. `STEP 04 / PROMPT ENGINE` — AI 优化提示词；
5. `STEP 05 / FINAL FRAME` — 生成高清图片。

视觉：

- 一条细长的垂直投影光束贯穿流程；
- 步骤沿光束左右交替排布；
- 每一步配一张真实电影静帧或创作过程影像；
- 左下可出现低对比投影机局部轮廓，不能用 CSS 手绘代替真实资产。

动画：

- 光束根据章节可见进度从上向下展开；
- 步骤节点依次点亮；
- 图片只做 1.02–1.04 的缓慢缩放和轻微位移；
- 移动端改为左侧单线时间轴，所有内容在右侧垂直排列。

### 5.5 第四屏：作品 Gallery / FRAME 04

标题：

`AI 镜头库 / 电影级作品示例`

作品标签：

- `Cinematic Portrait`
- `Luxury Product Shot`
- `Dark Fantasy Scene`
- `Editorial Fashion`
- `Architectural Vision`
- `Cinematic Car Scene`

布局：

- 使用 6 张真实生成图片；
- 桌面端采用不规则编辑杂志布局，图片大小与纵横比错落；
- 图片依附于可识别的基线网格，避免随机散乱；
- 移动端改为两列拼贴，重点人像可跨两列；
- 不使用九宫格或等宽卡片。

悬停与聚焦：

- 图片缓慢放大至 1.03；
- 暗色遮罩从 16% 增加到约 34%；
- 标题、类型与一句简短说明从底部淡入；
- 键盘聚焦提供与悬停同等信息；
- 触摸设备默认显示简短标签，不依赖悬停。

### 5.6 第五屏：积分与会员 / FRAME 05

标题：

`创作无界，灵感不设限`

权益：

- `新用户每日赠送 5 个积分`
- `每生成 1 张图片消耗 1 个积分`
- `积分不足时可升级解锁更多创作次数`

视觉：

- 以私人放映邀请或创作通行证的方式呈现；
- 三项权益使用银白图标、细线分隔和宽阔留白；
- 不出现月付价格、推荐套餐、折扣标签或三列价格卡；
- 背景可使用导演椅、放映厅或影像墙的低对比局部。

交互：

- `升级创作权限` 打开前端占位弹窗；
- 弹窗说明升级功能即将开放，不创建订单、不接入支付；
- 弹窗可通过关闭按钮、遮罩和 Escape 关闭；
- 打开时锁定背景滚动，并将焦点移动到弹窗。

### 5.7 第六屏：最终 CTA / FRAME 06

文案：

```text
每一次生成，
都是一帧电影。

开始生成你的第一张作品
```

视觉：

- 接近纯黑的空场；
- 单一顶光照亮中心区域；
- 背景只有极轻的颗粒、雾和光流；
- 不设置复杂 Footer，仅在底部保留品牌、版权和必要链接。

交互：

- CTA 进入 `/generate`；
- 光影缓慢流动，但幅度很小；
- 章节进入时文字逐行淡入。

## 6. 影像资产

页面不得使用占位框、CSS 插画、手绘 SVG 场景或无关图库拼凑。实现阶段使用 ImageGen 生成并保存以下项目资产：

1. Hero 巨幕：暗色奇幻电影场景，带人物背影和远处建筑；
2. 核心能力背景：黑色展厅、光幕与行走人物；
3. 流程氛围：投影机或摄影机局部、光束和烟尘；
4. Gallery 六张作品：人像、香水或高级产品、暗黑幻想、编辑时装、建筑、雨夜汽车；
5. 会员背景：导演椅或私人放映厅；
6. 最终 CTA 纹理：纯黑空间中的单一顶光。

资产规则：

- 输出 WebP 或 AVIF，并提供合理的 `sizes`；
- 首屏图优先加载，其余图片延迟加载；
- 保留每张图的视觉焦点，移动端使用独立 `object-position`；
- 所有图片具备描述性替代文本；
- 不在图片内依赖生成式文字。

## 7. 技术结构

使用既有规划中的技术方向：

- Next.js App Router；
- React；
- TypeScript；
- Tailwind CSS；
- Lucide React 提供常规界面图标；
- 不引入 GSAP、Three.js 或重量级滚动动画框架。

建议组件：

```text
app/
  page.tsx
  generate/page.tsx
  globals.css
components/
  cinematic/
    cinematic-header.tsx
    hero-section.tsx
    capabilities-section.tsx
    process-section.tsx
    gallery-section.tsx
    membership-section.tsx
    final-cta-section.tsx
    reveal.tsx
    film-grain.tsx
    upgrade-dialog.tsx
  ui/
    cinematic-button.tsx
lib/
  landing-content.ts
public/
  cinematic/
```

边界：

- 页面文案和 Gallery 元数据集中在 `lib/landing-content.ts`；
- 服务端组件负责静态结构和图片；
- 仅导航、弹窗、可见性动画和视差辅助组件使用客户端代码；
- `/generate` 在本轮提供一个可运行的同风格入口页，明确标注创作工作台将在后续接入；不实现真实生图流程。

## 8. 动效方案

### 8.1 出场动画

使用 `IntersectionObserver` 为元素添加可见状态：

- 初始：`opacity: 0`，`translateY(20px)`；
- 进入：`opacity: 1`，`translateY(0)`；
- 时长：700–1100ms；
- 缓动：`cubic-bezier(0.22, 1, 0.36, 1)`；
- 同组元素以 80–140ms 错开。

### 8.2 视差

- 仅 Hero 巨幕、人物剪影和少量背景光层参与；
- 位移范围不超过视口高度的 5%；
- 使用 `requestAnimationFrame` 合并滚动更新；
- 不监听每个元素的独立高频滚动事件；
- 移动端将位移降低一半。

### 8.3 减少动态效果

当用户启用 `prefers-reduced-motion: reduce`：

- 禁用视差和持续流光；
- 出场动画缩短为 120ms 淡入；
- 禁用图片缩放；
- 保留所有信息和交互。

## 9. 响应式

### 桌面端：`>= 1024px`

- 章节最小高度约为 90–110svh；
- Hero 使用左右非对称布局；
- 流程区使用交替式垂直时间轴；
- Gallery 使用 12 列编辑网格；
- 大量留白和超大标题。

### 平板：`768–1023px`

- 保留巨幕感，但减少透视角度；
- 流程图片统一到光束右侧；
- Gallery 使用 6 列网格；
- 导航可切换为菜单。

### 移动端：`< 768px`

- 页面自然垂直滚动，不使用横向溢出；
- 每屏保持 75–100svh 的海报感，但内容较多时允许自然增高；
- Hero 巨幕位于文案下方或作为全宽背景；
- 能力和流程全部单列；
- Gallery 两列拼贴，重点图跨列；
- 按钮最小触控高度为 48px；
- 文案不能被固定高度裁切。

## 10. 可访问性与性能

- 正文与背景对比度达到 WCAG AA；
- 所有按钮、链接、菜单和弹窗可通过键盘操作；
- 焦点样式使用细冷青描边，不依赖颜色变化之外的单一提示；
- 使用语义化 `header`、`nav`、`main`、`section`、`footer`；
- 每个章节具有唯一标题和可用的锚点；
- 首屏主要内容优先渲染，动画失败时页面仍完整可读；
- 避免布局偏移，图片声明尺寸或比例；
- 目标：桌面和移动端无横向溢出，滚动持续流畅；
- 目标：生产构建通过，首屏图之外不进行不必要的预加载。

- Header 锚点正确滚动；
- 两处“开始创作”均进入 `/generate`；

- 移动菜单可打开、关闭并正确管理焦点；
- 升级弹窗可通过按钮、遮罩和 Escape 关闭；

- `prefers-reduced-motion` 下无视差和持续动画。

### 视觉验收

- 1440px 桌面截图与选定视觉稿并排比较；
- 390px 移动端截图检查竖版海报感；
- 检查图片裁切、标题换行、细线、间距、光效和黑阶；
- 页面不能出现普通 SaaS 卡片堆叠、廉价渐变或紫蓝霓虹；
- 六个章节在快速浏览时仍可被清楚区分。

### 工程验收

- lint 通过；
- TypeScript 类型检查通过；
- 组件测试通过；
- 生产构建通过；
- 无控制台错误；
- 桌面与移动端均无横向滚动条。

## 12. 错误与降级

- 图片加载失败时保留黑色比例容器、可读标签和替代文本；
- JavaScript 不可用时，页面仍展示全部内容，仅失去动画和弹窗增强；
- 不支持 `backdrop-filter` 时使用更高不透明度黑色背景；
- 字体加载失败时使用规定的系统字体回退；
- 升级入口始终只展示“即将开放”，不得产生支付或订单状态。

## 13. 非目标

本轮不实现：

- 真实 AI 生图接口；
- 登录、注册或用户会话；
- Supabase 数据库和 Storage；
- 积分扣减或每日积分后台逻辑；
- 真实会员套餐和支付；
- 历史记录、收藏或下载；
- CMS 或后台配置；
- WebGL、3D 场景和音频播放。

这些能力继续沿用原绘境 AI MVP 规格，后续可在相同品牌视觉下接入。
